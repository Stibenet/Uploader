
Функциональная потребность:
Иметь на ПК пользователя актуальную (последнюю) версию ПО.

Прецеденты:
1. Загрузка в СУБД обновления ПО.
2. Выгрузка из СУБД текущей версии ПО.

Программные требования:
1. Загрузить в СУБД несколько файлов.
2. Нарастить версию ПО, хранимого в СУБД.
3. Сверить версию ПО в СУБД с локальной версией ПО.
4. Выгрузить из СУБД несколько файлов.
5. Задать версию локалького ПО равной версии ПО в СУБД.

Дополнительно:
6. Необходимо иметь возможность обновить ЛЮБОЙ файл приложения, в том числе
	запускной.

-------------------------------------------------------------------------------

Прецедент "Загрузка в СУБД обновления ПО":
1. Инженер службы сопровождения запускает загрузчик.
2. Загрузчик отображает UI для загрузки файлов в СУБД.
	- Строка подключения к СУБД извлекается из конфигурационного файла.
	- Версия ПО задается как большая на единицу текущей серверной версии.
3. Инженер указывает (1) перечень файлов, которые необходимо загрузить в СУБД,
	возможно, (2) версию ПО и инициирует загрузку.
4. Загрузчик (1) загружает в СУБД все указанные файлы и (2) задаёт новую версию
	серверного ПО.

Прецедент "Выгрузка из СУБД текущей версии ПО":
1. Пользователь запускает приложение.
2. Приложение сверяет свою версию с версией ПО, хранимого в СУБД. Выяснив, что
	локальная версия младше серверной, ПО выгружает из СУБД загрузчик, запускает
	его и завершается. Загрузчик (1) выгружает из СУБД все файлы приложения,
	(2) задаёт версию локального ПО равной серверной, (3) запускает приложение и
	завершается.
